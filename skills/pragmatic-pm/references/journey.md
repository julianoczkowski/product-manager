# The Pragmatic PM journey — stages, skills, and artifacts

Each stage produces one artifact. A PM rarely does all of them at once — the router
picks the right entry point for what the user actually asked, then offers the next step.

## Routing "I want to build a feature"

This is the most common entry point. It is NOT "write me a PRD." A market-driven PM
works it in this order — offer to run it as a guided sequence, or jump to any stage:

1. **What problem, for whom?** → `market-problems` (+ `personas`) — confirm the problem is urgent, pervasive, willing-to-pay. Don't skip this; it prevents building the wrong thing.
2. **Can we win / should we?** → `competitive-landscape`, `distinctive-competence`, `opportunity-scoring`.
3. **Make the case** → `product-canvas` (business case, buy/build/partner, market sizing).
4. **Position it** → `positioning`.
5. **Specify it** → `requirements` + `use-scenarios` → assemble into a `prd`.
6. **Plan & commit** → `product-roadmap` → `release-plan`.
7. **Align & launch** → `stakeholder-communication` → `launch-plan`.

If the user is impatient or the feature is small, offer the **fast path**: `market-problems`
(quick) → `prd` → `release-plan`, and note which validation steps were skipped so they're
aware of the risk.

- "competitors / competitive analysis / battlecard" → `competitive-landscape`
- "persona / who is the buyer / who is the user" → `personas`
